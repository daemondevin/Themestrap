PluginPopover — Live Demo

Accessible, anchor-positioned popovers — four sides, three alignment modes, viewport flip, portaling, CSS arrows, side-aware animations, and mutual exclusion. Zero build step.

Every popover below is a live instance of PluginPopover running directly in your browser. Click triggers to open them, then explore side / align variants with the buttons in Card 5.

Portaled popover

The orange dashed box has overflow: hidden. Without portaling the popover would be clipped. With portaling: true the content moves to <body> and positions absolutely against the viewport.

modal: true · focus trap · closeOnOutside: false

Modal popover (focus trap)

With modal: true Tab key cycles only inside the panel. closeOnOutside: false keeps it open while the user interacts with the form. Only Escape or the Cancel button closes it.

Table row action

The trigger lives inside a table cell; the popover wrapper is elsewhere in the DOM. Connected by matching the trigger's attribute value to the wrapper's id.
