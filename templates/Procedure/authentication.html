Authentication

Every request is authenticated with a bearer token. Here is how keys, scopes, and rotation work.

API keys

Procedure authenticates requests with API keys. Each key is either a publishable key (safe for client code) or a secret key (server-side only). Keys are scoped to a single environment.

PROTOCOL_KEY="sk_live_8f3a...e21"

Authenticated requests

Send the key as a bearer token on every request:

curl https://api.protocol.dev/v1/conversations \
  -H "Authorization: Bearer sk_live_8f3a...e21"
await fetch('https://api.protocol.dev/v1/conversations', {
  headers: { Authorization: `Bearer ${process.env.PROTOCOL_KEY}` }
})

Auth errors

Missing or invalid credentials return 401 Unauthorized:

{
  "error": {
    "type": "authentication_error",
    "message": "No API key provided."
  }
}
Create a second key, deploy it, then revoke the old one. Both keys stay valid during the overlap so there is no interruption.
