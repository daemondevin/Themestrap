Themestrap Module Loader

A tiny client-side orchestrator: you define() modules (each a JS file, a CSS file, dependencies, and an init hook), then require() them. It resolves the dependency graph, fetches each asset once, runs inits in order, and only settles after the DOM is ready. Below, everything is loaded lazily on click — nothing in the graph touches the network until you ask for it.

What each button proves

require(['confetti']) — walks the chain config → utils → ui → confetti in order, fetches a real library from cdnjs via $.getScript, then its init fires confetti. Click again: instant, because the module is in the loaded set.

require(['theme']) — lazy-loads animate.css and animates the box. The loader skips re-inserting a <link> that's already present.

scan() — reads [data-module] off the DOM and requires whatever it finds (no manual wiring).

runRoutes() — matches the current URL path against registered routes and loads the matching set.
