Create a resource

POST a JSON body to the resources endpoint to create a new item. The response includes the new resource ID and a confirmation timestamp.

Required fields: title (string), type (string). Optional: parent (integer), template (integer).

List resources

GET the resources collection. Supports query parameters: limit, offset, parent, and template.

Returns an array of resource objects with pagination metadata in the _meta key.

Update a resource

PATCH a resource by its ID. Only the provided fields are updated; omitted fields retain their current values.

A 404 is returned if the resource does not exist or the caller lacks edit permission.

Delete a resource

DELETE removes the resource permanently. Pass force=true to bypass the recycle bin.

A 204 No Content response is returned on success.

curl -X POST https://api.example.com/resources \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer TOKEN" \
  -d '{
    "title": "Getting Started",
    "type":  "document",
    "parent": 0
  }'

# Response
{
  "id": 42,
  "title": "Getting Started",
  "createdAt": "2026-07-18T14:00:00Z"
}
curl "https://api.example.com/resources?limit=10&parent=0" \
  -H "Authorization: Bearer TOKEN"

# Response
{
  "_meta": {{ "total": 42, "limit": 10, "offset": 0 }},
  "results": [
    {{ "id": 1, "title": "Home" }},
    {{ "id": 2, "title": "About" }}
  ]
}
curl -X PATCH https://api.example.com/resources/42 \
  -H "Authorization: Bearer TOKEN" \
  -d '{{"title": "Getting Started v2"}}'

# Response
{{
  "id": 42,
  "title": "Getting Started v2",
  "updatedAt": "2026-07-18T15:00:00Z"
}}
curl -X DELETE "https://api.example.com/resources/42?force=true" \
  -H "Authorization: Bearer TOKEN"

# Response: 204 No Content
