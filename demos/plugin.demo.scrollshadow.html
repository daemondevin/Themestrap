The mask color must match the container's background — pass it through color or set --ts-ss-color in CSS.

Themestrap was started as a way to stop rewriting the same handful of MODX snippets for every Bootstrap-based site I built. It started small — a tsCard, a tsAccordion — and grew into a full library of porting between Porto's HTML and MODX-native chunks.

The trick was getting the property metadata, chunk templates, and JavaScript plugin auto-init pipeline to all play nicely together. Once that landed, building new components became almost mechanical.

The hardest pieces were the bits where MODX's templating overlaps with Bootstrap's data-attribute conventions, like nav. pdoMenu turned out to be the right primitive once we threaded through unique instance IDs.

If you're considering using Themestrap on your own project, start with the snippets you'd rewrite anyway: alerts, cards, modals. Add plugins as you need them.

The next major release will add a server-side rendering pipeline for the JS plugins, so the initial HTML matches the post-init DOM. That should solve the FOUC issues some users have reported.
