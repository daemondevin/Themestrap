PluginNavmenu — Live Demo

shadcn/ui-style navigation menus — hover or click, animated mega panels, keyboard arrow navigation, edge-flip, viewport portal mode, mutual exclusion, and full ARIA. Zero build step.

Every nav below is a live PluginNavmenu instance — hover or click triggers to open panels. All keyboard interactions are active: ←→ to move between triggers, ↓ to dive into a panel, Esc to close.

openOn: hover · orientation: horizontal · mega panel

Standard site header nav

Drop-in for a typical top navigation. Hover (200 ms debounce) opens the panel; moving into the panel before the close timer fires keeps it open. The Products panel uses a featured card + 2-column list grid.

openOn: click · dark header · CSS custom properties

Click-to-open, dark header

Setting openOn: "click" makes every panel toggle on click — better for touch devices or keyboard-first users. Panels pick up dark colours via --ts-nav-content-bg and --ts-nav-content-border set on the parent element.

orientation: vertical · openOn: click · sidebar pattern

Vertical sidebar nav

Setting orientation: "vertical" switches to a flex-column layout. Panels slide out to the right of the trigger. Arrow keys ↑↓ move between triggers; → dives into an open panel.

Mutual exclusion · one panel open at a time

One panel at a time

Opening a new panel always closes the previously open one — no two panels stack. Try clicking each trigger in sequence. Switching between items with ←→ is instant (no exit animation between siblings).

Events · navmenu-opened / navmenu-closed · ARIA

Event log & ARIA

Both navmenu-opened and navmenu-closed are CustomEvents that bubble from the root [data-plugin-navmenu]. The detail carries index, $item, and instance. ARIA attributes update live — inspect the trigger in DevTools.
